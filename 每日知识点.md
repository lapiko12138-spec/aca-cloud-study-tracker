## Day 1 ｜ 2026-06-23 ｜ 云计算概念 + ECS 入门

### 一、核心概念 ✅
- **服务模型**
  - **IaaS**（基础设施即服务）：给你服务器/网络/存储，自己搭 → 例：ECS
  - **PaaS**（平台即服务）：给你运行环境/中间件 → 例：托管数据库
  - **SaaS**（软件即服务）：开箱即用的软件 → 例：在线 CRM、企业邮箱
- **部署模型**：公有云 / 私有云 / **混合云**（公私结合，按需分配业务）
- **地域 Region & 可用区 AZ**
  - 地域 = 不同城市的数据中心群；可用区 = 同地域内电力/网络独立的机房
  - 关键：**同地域多可用区延迟低 → 做高可用**；不同地域默认不内网互通
- **计费方式**：包年包月（长期稳定）/ 按量付费（短期·波动）/ 抢占式（最便宜，可能被回收）

### 二、核心产品速记表 🔑（重点，入学错题全在这）
| 缩写 | 全称 | 一句话 | 类比 |
|---|---|---|---|
| ECS | Elastic Compute Service | 云服务器，跑程序/网站 | 租来的电脑 |
| OSS | Object Storage Service | 对象存储，存图片/视频/备份 | 无限网盘 |
| RDS | Relational Database Service | 托管关系型数据库(MySQL等) | 别人运维的数据库 |
| SLB | Server Load Balancer | 负载均衡，分发流量 | 餐厅领位员 |
| VPC | Virtual Private Cloud | 专有网络 | 院墙/内网 |
| RAM | Resource Access Management | 访问控制，管权限 | 门禁卡 |
| CDN | Content Delivery Network | 内容分发，就近加速 | 连锁便利店 |
| NAS | Network Attached Storage | 文件存储，多机共享 | 共享文件夹 |

### 三、ECS 五段式精讲 ⭐（考试占比最高）
1. **概念**：阿里云机房里你独享的一台虚拟服务器，可选 CPU/内存/系统/硬盘，远程登录上去跑程序。
2. **为什么存在**：替代自购物理服务器——分钟级开通、按需付费、随时增减、宕机自动迁移；把"重资产采购"变"轻运营租用"。
3. **实际案例**：电商双 11 用弹性伸缩临时扩 100 台 ECS 扛流量，大促后自动释放，只为高峰那几天付费。
4. **高频考点**：ECS=IaaS/弹性计算；镜像=系统模板；快照=磁盘备份；安全组=虚拟防火墙；三种计费；规格族(通用/计算/内存/GPU 型)。
5. **面试提问**："ECS vs 自购服务器省在哪？" → 省钱(免硬件 CapEx)·省心(运维阿里云管)·弹(随业务增减)·稳(自动迁移+多 AZ 高可用)·快(分钟级上线)。

### 四、课时1 PPT 精华 📷（你的截图整理）

**① 传统 IT（自建机房）要操心的事 → 云为什么出现**
- 机房(电力/空调/机柜) + 硬件(服务器/交换机/路由器/防火墙/IPS/IDS) + 软件(操作系统/Web服务器/数据库) + 人员(施工/维护/DBA/系统/网络管理员)
- 痛点：**耗时耗力、人+设备成本高、不利于扩容**
- 本质：**传统IT＝购买硬件→部署硬件→部署软件→部署应用；云计算＝直接购买服务**
- 🔌 经典类比：像电力——100 年前各家自发电，今天直接用电网。云＝计算界的"电网"（计算/网络/存储三大资源池）

**② 部署模式（按"谁拥有、谁使用"分）**
- 专有云（＝私有云）：企业内部自建 ｜ 公有云：服务商提供、经互联网用 ｜ 混合云：内外互通，兼顾私密性+灵活性
- ⚠️ 术语坑：阿里云把"私有云"叫**专有云**

**③ 服务模式（可操控性 IaaS > PaaS > SaaS）**
- IaaS 基础设施即服务：给计算/存储/网络资源，自己搭系统，**可操控性最高**（ECS 即典型）
- PaaS 平台即服务：给开发者按需开发/测试环境
- SaaS 软件即服务：直接用厂商软件（如在线 CRM）

**④ 阿里云简介（高频记忆点）**
- 2009 年成立；服务 **200+ 国家和地区**；**2017.1 成为奥运会全球指定云服务商**
- 全球数据中心：中国华东、华北各约 15 个可用区（印证"一个地域含多个可用区"）

> 🎯 考点：①云＝购买服务 ②三种部署模式定义 ③IaaS/PaaS/SaaS 区别 +「可操控性最高＝IaaS」④专有云＝私有云
> 💼 销售话术：客户嫌上云贵 → 用"自建机房要养机房+硬件+软件+一堆人、还扩容慢"对比，云像用电一样按需买。

**六大产品类别（先记"哪个产品属于哪一类"＝治你错题）**
| 类别 | 旗舰产品 |
|---|---|
| 弹性计算 | ECS、弹性裸金属(神龙)、GPU/FPGA、弹性伸缩、资源编排 |
| 存储 | 对象存储 OSS、块存储、文件存储 NAS、归档存储 |

**🔥 存储三兄弟（高频考点 + 你 Q5 错因）**
- **OSS 对象存储**：海量非结构化大文件(图片/视频/备份)，比传统存储省 25%~75% → "存了不常改"
- **块存储**：块级**随机**读写、高性能高可靠 → 给 ECS 当系统盘/数据盘 ← ⚠️"随机读写"是它，不是 OSS！
- **NAS 文件存储**：支持 NFS/CIFS 协议，多台机器**共享**文件

**弹性计算补充**
- 神龙(弹性裸金属)：**虚拟机的弹性 + 物理机的性能**、物理隔离、分钟交付 ⭐
- 弹性伸缩：自动增减计算资源（双 11 自动扩容就靠它）

**网络补充**
- VPC＝逻辑隔离网络 ｜ SLB＝多台 ECS 流量分发 ｜ NAT 网关＝转发/共享带宽 ｜ EIP＝独立公网 IP ｜ 云企业网＝跨地域企业级互联 ｜ VPN 网关＝云上云下加密互通

> 🎯 考点：①产品归类（哪个属于哪类）②存储三兄弟区别 ③神龙特点
> 💼 销售：PolarDB 是阿里**自研**关系型数据库（对标 Oracle / AWS Aurora）；神龙卖给"既要性能又要弹性"的客户
> 📌 注意：**RAM（你 Q9 错题）不在产品全家福里**——它是"访问控制/账号权限"，属安全管理，阶段三讲

### 六、课时4–5 PPT 精华：数据库 + 体系架构 + 地域可用区 📷

**数据库类产品（你 SQL 背景的主场）**
- **RDS**：托管关系型数据库；RDS MySQL 版最常用（LAMP 经典组合里的 M）
- **PolarDB**：阿里**自研**云数据库，兼容 MySQL/PostgreSQL、兼容 Oracle 语法，性能最高达传统 MySQL **6 倍**，存储可达 100T —— 💼 卖点：高性能+兼容性，替换 Oracle 降本
- **DTS** 数据传输服务：数据库间**迁移/同步/订阅**（上云搬数据靠它）
- **DMS** 数据管理：网页版数据库管理工具（比 Navicat 易用）

**体系架构（了解即可，注意老名词）**
- 底层：数据中心 → Linux → 分布式文件系统/任务调度/资源管理 → 上层 ECS/OSS/RDS/ODPS/OTS
- ⚠️ 老 PPT 名词：**ODPS ＝ 现在的 MaxCompute**（大数据）、**OTS ＝ 现在的 TableStore**（结构化）——你做数据分析/AI 会用到 MaxCompute

**🔥 地域 Region & 可用区 Zone（高频考点，必背）**
- 地域 ＝ 按地理位置划分；**资源创建后不能修改地域**（⚠️ 高频陷阱题）；不同地域资源略有差异
- 可用区 ＝ 同地域内**电力/网络独立**的物理区域；同可用区内实例延时更小
- 关系：**地域完全独立；可用区完全隔离，但同地域内可用区之间低时延互联**
- 选地域考虑：①目标用户地理位置 ②产品关系 ③价格 ④**备案**（国内业务，中国特色考点）
- 选可用区考虑：**容灾能力 ↔ 网络延时** 权衡（要容灾就跨可用区，要低延时就同可用区）

> 🎯 必考：①地域创建后不可改 ②可用区＝故障隔离+低时延互联（→多可用区做高可用）③选地域要备案
